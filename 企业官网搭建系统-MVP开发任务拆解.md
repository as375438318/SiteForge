> 本文把 16 周 MVP 开发计划拆解为可认领的工程任务卡。每个任务含 ID、描述、负责角色、预估工时（人天）、前置依赖、验收标准。团队可直接基于此排 Sprint。

---

## 任务编号规则

```
SF-<阶段>-<模块>-<序号>

阶段：INF(基础设施) | PARSE(解析) | SSG(静态生成) | EDIT(编辑器)
      CMS(内容管理) | SEO | GEO | FORM(表单) | INTEG(集成测试)
模块：对应架构方案中的子系统
```


### SF-INF-LIC-01 License 授权系统

**描述**：实现 License 系统：① 本地机器码生成（CPU+MAC+磁盘序列号）② 云端 License 签发服务（RSA 私钥签名）③ 本地 License 验证模块（RSA 公钥验签 + machineId/domain/过期校验）④ License 失效时锁定管理功能但保持静态站可访问。

**验收标准**：

- [ ] RSA 签名验证通过；篡改任意字段后验签失败
- [ ] License 过期 → 管理后台不可登录，但已发布静态站正常访问
- [ ] License 绑定错误 machineId → 验证失败
- [ ] 后台显示 License 状态（有效/过期/无效）+ 到期提醒

## 2. W3-W4：AI 结构解析引擎（PARSE）

### SF-PARSE-CLOUD-01 云端解析服务搭建

| 字段 | 内容 |
|------|------|
| 角色 | 解析引擎工程师 |
| 工时 | 4 人天 |
| 前置 | 无（可与基础设施并行） |
| 依赖产出 | 云端解析 API 服务 |

**描述**：搭建云端解析服务（独立部署，我方运维），接收 URL，返回结构化 JSON。包含 Playwright 渲染、cheerio DOM 提取、robots.txt 合规检查。

**验收标准**：
- [ ] API 接收 POST `{ url }`，返回 `{ pages: [...], navigation: [...] }`
- [ ] 先检查 robots.txt，Disallow 路径返回拒绝
- [ ] Playwright 渲染页面（等待 networkidle，超时 30s 降级为纯 HTTP）
- [ ] 限频：单 URL 缓存结果，同站 QPS ≤ 1
- [ ] 处理失败返回明确错误码：UNREACHABLE / BLOCKED / TIMEOUT / PARSE_FAILED
- [ ] API 需鉴权（API Key，防止滥用）
